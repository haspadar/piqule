# piqule

Piqule manages quality-tool configuration and runs quality checks for this project. All files under `.piqule/` and piqule-managed files under `.github/` are generated from templates and must not be edited by hand.

## Commands

- `vendor/bin/piqule check` — run all enabled checks.
- `vendor/bin/piqule check <tool>` — run a single check (for example `phpstan`, `psalm`, `phpunit`, `phpcs`, `phpmd`, `infection`).
- `vendor/bin/piqule fix` — run all auto-fixers.
- `vendor/bin/piqule sync` — regenerate `.piqule/` and `.github/` from templates and re-pin template checksums.

Always invoke quality tools through `vendor/bin/piqule check <tool>`, not through `vendor/bin/phpcs`, `vendor/bin/phpstan`, etc. — the wrapper applies the project-level configuration.

## Timeouts

- `vendor/bin/piqule check` — fits the default command timeout.
- `vendor/bin/piqule check sonar` and `vendor/bin/piqule check full` — allow up to 5 minutes.

## Configuration

- Project settings live in `.piqule.yaml` at the repository root (optional).
- Keys are flat and dot-separated (for example `ci.php.matrix`), never nested arrays.

## Do not edit

- `.piqule/**` — regenerated by `piqule sync`.
- `.github/workflows/**` that piqule owns — regenerated by `piqule sync`.
- To change these files, edit the source templates (or piqule configuration) and run `vendor/bin/piqule sync`.

## Suppressions

Do not add suppress directives (`// NOSONAR`, `@phpstan-ignore`, `@phpstan-ignore-next-line`, `@psalm-suppress`, baseline entries) to silence findings. Fix the underlying issue instead.
